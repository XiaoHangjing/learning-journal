## Content Insights

This week helped me understand more clearly how HTML, CSS and JavaScript work together in a webpage.

HTML and CSS were already quite clear to me before. I understood HTML as the structure of the page, such as headings, images, buttons and sections, while CSS is used for styling like layout, colours and spacing.

I didn’t really understand JavaScript before. I used to think that changing pages was just a matter of adding a link or a button that opens another webpage. I didn’t realise JavaScript has a much bigger role in controlling behaviour and interaction on the page.

After working on this project, I started to understand that JavaScript is not just for navigation, but for controlling how elements respond to user actions, such as scrolling effects and animations. This changed my understanding of how a webpage actually functions, especially when combining HTML, CSS and JavaScript together.

Bootstrap was also something I focused on this week. I didn’t really understand how writing a class could replace so much CSS, but after using it a few times, it made more sense. For example:

```html
<div class="card p-3 shadow">
```

This is much shorter than writing everything manually in CSS, but it still gives you a clean styled box. It feels more like combining pre-made “building blocks” instead of starting from zero.
The main thing I learned this week was the scroll animation effect. I managed to make cards appear and move upwards when scrolling down the page. I don't really understand how it works, but after testing it a few times, I realized it's a combination of CSS transform and opacity changes implemented in JavaScript
﻿
CSS handles the animation itself (opacity and moving position), while JavaScript decides when to trigger it based on sroll position. That separation made more sense after actually building it.
I also learned how to make the navigation bar fixed at the top of the page and how to link it to different pages. I only knew how to create basic links between pages, but after trying it in the project, I understood how the navigation bar stays visible while scrolling using Bootstrap classes like fixed-top. This made the website easier to navigate, and I can see how this is commonly used in real websites to improve user experience.
I also used GitHub Pages to publish my website.I only used GitHub to store files, but now I understand it can actually host a working website that people can open with a link.

## Career/Employability/Learning Insights

This week made me realise that front-end development is more about understanding patterns than memorising code.

Bootstrap in particular showed me how real developers probably don’t write everything from scratch. Instead, they use frameworks to save time and keep things consistent. 

The scroll animation part was probably the most interesting for me. It took me a while to understand why the effect works, and I had to change things a few times before it looked right. This made me realise I learn better when I actually experiment instead of just watching tutorials.

Using GitHub Pages also gave me a better idea of how websites are actually published. It feels different when your code is no longer just local files but something people can open online.

For now, I’m still more comfortable with HTML and CSS. JavaScript still feels a bit unclear, especially when it comes to deeper logic like DOM manipulation. In the next few weeks, I plan to explore and learn more page effects and interactive features to better understand how websites become more dynamic.
